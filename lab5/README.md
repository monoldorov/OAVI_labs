# Лабораторная работа №5. Выделение признаков символов. Вариант 14

**Испанские заглавные буквы.**
**Алфавит:** `ABCDEFGHIJKLMNÑOPQRSTUVWXYZ`

## Теоретическая часть

В данной лабораторной работе рассматривается выделение признаков бинарных изображений символов. По условию лабораторной для бакалавра требуется сгенерировать эталонные изображения символов выбранного алфавита, обрезать белые поля, вычислить набор признаков и сохранить результаты в файлы. Для варианта 14 используются испанские заглавные буквы.

В лекции 5 рассматриваются признаки бинарных изображений, среди которых:

вес и удельный вес;
координаты центра тяжести;
осевые моменты инерции;
профили X и Y.

**Вес** — это количество пикселей объекта. В данной работе считается именно масса чёрного, как требует условие лабораторной.
**Удельный вес** — это вес, нормированный к площади рассматриваемой области.
**Центр тяжести** показывает, где сосредоточена основная масса символа.
**Осевые моменты инерции** позволяют оценить распределение массы относительно горизонтальной и вертикальной осей.
**Профили X и Y** — это суммы чёрных пикселей по столбцам и строкам соответственно. Они являются важными признаками формы символа.

**По условию лабораторной:**

- символы должны генерироваться программно;
- белые поля желательно сразу обрезать;
- изображения должны сохраняться по принципу «1 символ — 1 файл»;
- скалярные признаки должны сохраняться в CSV с разделителем ;;
- профили X и Y должны сохраняться в PNG как столбчатые диаграммы с корректной ориентацией и целыми числами на осях.

## Практическая часть
Принцип работы программы

Программа работает следующим образом:

- Загружается системный .ttf-шрифт.
- Для каждого символа испанского алфавита создаётся отдельное изображение.
- Символ рисуется на белом фоне.
- Изображение переводится в бинарный вид.
- Белые поля вокруг символа автоматически обрезаются.
- Обрезанный символ сохраняется в отдельный PNG.
- Для символа вычисляются скалярные признаки.
- Для символа вычисляются профили X и Y.
- Профили сохраняются в виде столбчатых диаграмм PNG.
- Все скалярные признаки сохраняются в общий CSV-файл.

Что реализуется в файлах
**config.py** 

Содержит настройки:

- алфавит;
- размер шрифта;
- размер холста;
- порог бинаризации;
- пути сохранения результатов;
- список возможных путей к системным шрифтам.

**font_utils.py**

Отвечает за поиск и загрузку шрифта.
Если шрифт не найден, выбрасывается понятная ошибка.
Это сделано для того, чтобы Pillow не перешёл на встроенный маленький bitmap-шрифт.

**image_features.py**

Содержит основную обработку:

- генерацию символов;
- бинаризацию;
- обрезку белых полей;
- вычисление признаков;
- вычисление профилей;
- сохранение изображений и диаграмм.

**main.py**

Главный файл запуска:

- создаёт папки;
- загружает шрифт;
- обрабатывает весь алфавит;
- сохраняет символы, профили и таблицу признаков.

## Описание обработки
### Часть 1. Генерация символов

Используется алфавит:

ABCDEFGHIJKLMNÑOPQRSTUVWXYZ

Символы не берутся из готовых изображений, а генерируются программно через Python с использованием системного .ttf-шрифта. Это соответствует условию лабораторной.

Что формировалось

Для каждого символа автоматически создавалось отдельное бинарное изображение:

- символ рисовался на белом фоне;
- затем выполнялась бинаризация;
- затем белые поля обрезались.

**Куда сохранялись результаты** 
Готовые символы сохранялись в папку:

`output/symbols/`

Примеры имён файлов:

A.png
B.png
C.png
...
Ñ.png
...
Z.png

**Особенности обработки**

Белые поля действительно обрезаются сразу после бинаризации.
После обрезки символ не помещается обратно на большой холст.
Соблюдается принцип «1 символ — 1 файл».
Символ Ñ поддерживается отдельно и сохраняется как самостоятельный файл.

**Вывод по части 1**

На этом этапе были получены эталонные бинарные изображения всех символов испанского заглавного алфавита. Обрезка белых полей делает изображения компактными и более удобными для дальнейшего анализа.

### Часть 2. Вычисление скалярных признаков
Что формировалось

Для каждого символа вычислялись следующие скалярные признаки:

- масса чёрного каждой четверти изображения;
- удельная масса каждой четверти;
- координаты центра тяжести;
- нормированные координаты центра тяжести;
- осевые моменты инерции по горизонтали и вертикали;
- нормированные осевые моменты инерции.

**Как выполнялась обработка**

После обрезки изображение символа делилось на 4 четверти:

- верхняя левая;
- верхняя правая;
- нижняя левая;
- нижняя правая.

Для каждой четверти считалось количество чёрных пикселей.
Именно эта величина использовалась как масса чёрного.
Затем масса нормировалась к площади четверти.

Координаты центра тяжести рассчитывались по всем чёрным пикселям изображения.
Осевые моменты инерции вычислялись относительно центра тяжести по горизонтальной и вертикальной осям.

**Куда сохранялись результаты**

Все скалярные признаки сохранялись в файл:

`output/scalar_features.csv`

**Особенности сохранения**
- используется формат CSV;
- есть строка заголовков;
- разделитель — ;;
- целочисленные признаки по возможности сохраняются как целые числа, без .0.

**Вывод по части 2**

Скалярные признаки позволяют представить каждый символ как набор числовых характеристик. Это даёт возможность сравнивать символы не только визуально, но и количественно.

### Часть 3. Профили X и Y
**Что формировалось**

Для каждого символа вычислялись:

- профиль X — сумма чёрных пикселей по каждому столбцу;
- профиль Y — сумма чёрных пикселей по каждой строке.
**Как выполнялась обработка**

Профиль X показывает распределение массы символа по горизонтальной координате.
Профиль Y показывает распределение массы символа по вертикальной координате.

После вычисления профили сохранялись в виде столбчатых диаграмм.

**Куда сохранялись результаты** 

Профили X сохранялись в папку:

`output/profiles_x/`

Профили Y сохранялись в папку:

`output/profiles_y/`

Примеры имён файлов:

A_profile_x.png
A_profile_y.png
Ñ_profile_x.png
Ñ_profile_y.png
**Особенности обработки**

Профили сохраняются именно как столбчатые диаграммы.
Ориентация профилей правильная.
Подписи осей сделаны корректно.
На осях используются целые числа, как требует условие лабораторной.

**Вывод по части 3**

Профили X и Y дают наглядное представление о распределении массы символа по строкам и столбцам. Они хорошо дополняют скалярные признаки и позволяют визуально сравнивать форму разных букв.

## Автоматизация сохранения результатов

Программа не требует ручного задания имён для результатов.
Для каждого символа автоматически формируются:

- изображение символа;
- профиль X;
- профиль Y;
- строка в общем CSV.

Результаты автоматически раскладываются по нужным папкам:

`output/symbols/`
`output/profiles_x/`
`output/profiles_y/`
`output/scalar_features.csv`

Это позволяет:

- не переименовывать файлы вручную;
- удобно хранить результаты;
- быстро сопоставлять изображения символов и их признаки.

В лабораторной работе была реализована программа для выделения признаков символов испанского заглавного алфавита в соответствии с вариантом 14. Символы генерировались программно, переводились в бинарный вид, автоматически обрезались по содержимому и сохранялись по принципу «1 символ — 1 файл».
